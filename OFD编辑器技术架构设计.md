# OFD编辑器技术架构设计（V1）

## 1. 架构目标

- 标准兼容：严格遵循 OFD 标准结构，确保互通。
- 业务可用：满足编辑、审阅、签章、归档全链路。
- 平台可扩：支持服务化、私有化、二次开发。
- 安全可管：权限、水印、审计全流程可追踪。

## 2. 总体架构

- 端侧（Desktop/Web）
  - 文档交互层：阅读、编辑、审阅、签章操作 UI。
  - 文档内核层：解析、渲染、编辑模型、增量保存。
  - 安全插件层：证书管理、签章调用、本地策略执行。

- 服务端（Document Service）
  - 文档处理服务：转换、批量任务、页面处理。
  - 签章验章服务：统一签章策略、证书链校验、结果回传。
  - 安全治理服务：水印策略、权限模板、审计中心。
  - 集成网关：OA/档案/签章平台对接 API。

## 3. 模块分解与职责

## 3.1 Core-Parser（解析模块）
- 负责 OFD 包解析、对象树构建、资源索引、结构校验。
- 输出标准对象模型（DOM-like）供渲染与编辑复用。
- 支持增量更新与损坏文档容错修复（可配置）。

## 3.2 Core-Render（渲染模块）
- 负责页面布局计算、矢量绘制、文本排版、资源缓存。
- 支持高 DPI 缩放与分页渲染，保证所见即所得。
- 提供渲染快照给打印与导出模块复用。

## 3.3 Core-Edit（编辑模块）
- 负责对象级编辑命令（插入、删除、替换、移动、对齐）。
- 提供命令栈（Undo/Redo）、差异补丁（Delta）与冲突检测。
- 对外暴露统一编辑 API，屏蔽底层文档细节。

## 3.4 Review（审阅模块）
- 注释、修订、作者维度筛选、批注导入导出。
- 审阅状态机：待处理/接受/拒绝/已归档。
- 生成审阅报告用于流程系统回填。

## 3.5 Sign（签章模块）
- 盖章策略：自由盖章、关键字定位、骑缝章、批量盖章。
- 验章流程：签名验证 -> 证书链检查 -> 完整性校验 -> 状态输出。
- 对接外部签章服务时提供适配器层（Provider Adapter）。

## 3.6 Convert（转换模块）
- 首期支持 OFD/PDF 双向转换，P1 增加 Office 族支持。
- 统一任务模型：提交、排队、执行、重试、回调。
- 提供异常归因（字体缺失、资源损坏、对象不兼容）。

## 3.7 Security（安全模块）
- 策略中心：打印/复制/导出等权限控制策略。
- 水印引擎：静态、动态、可追踪水印规则。
- 审计日志：操作主体、动作、对象、时间、结果全量记录。

## 3.8 Integration（集成模块）
- REST API：文档处理、签章验章、任务查询、审计检索。
- SDK 层：Java/C++/.NET，保证关键能力同构输出。
- 回调机制：支持异步任务状态通知与错误码标准化。

## 4. 关键数据模型

- `DocumentModel`：文档元数据、页面树、资源集合、版本号。
- `EditDelta`：编辑差异记录（操作类型、对象 ID、前后状态）。
- `SignRecord`：签章位置、证书标识、时间戳、验章状态。
- `TaskRecord`：任务类型、参数、进度、重试次数、结果摘要。
- `AuditEvent`：主体、行为、对象、来源、结果、追踪 ID。

## 5. 接口设计（示例）

- `POST /api/v1/convert`
  - 入参：源文件、目标格式、策略参数
  - 出参：`taskId`

- `POST /api/v1/sign`
  - 入参：文档 ID、签章策略、证书标识
  - 出参：签章结果与新版本文档 ID

- `POST /api/v1/verify`
  - 入参：文档 ID
  - 出参：签章列表、证书状态、完整性结果

- `GET /api/v1/tasks/{taskId}`
  - 出参：任务状态、进度、失败原因、产物地址

## 6. 技术选型建议

- 客户端框架：C++/Qt（桌面）或 Electron（跨平台快速迭代）。
- 渲染与图形：Skia/Cairo 类图形后端（按团队经验选择）。
- 服务端：Java/Spring Boot 或 Go（高并发任务服务）。
- 存储：
  - 对象存储：文档原件与中间产物
  - 关系库：任务、审计、策略元数据
  - 缓存队列：任务调度与状态加速

## 7. 部署形态

- 单机版：客户端 + 本地轻服务（适合离线场景）。
- 部门版：私有化部署，集中策略与审计。
- 集团版：多租户 + 多节点任务集群 + 统一网关。

## 8. 质量与测试体系

- 兼容测试：标准样本、行业样本、异常样本三套回归。
- 性能测试：大文档打开、批量转换、并发签章压测。
- 安全测试：权限绕过、水印失效、日志篡改检测。
- 稳定性测试：长时任务、断点续传、失败重试可靠性。

## 9. 里程碑与交付物

- P0 交付
  - 编辑器 MVP
  - OFD/PDF 转换服务
  - 单章签章与验章

- P1 交付
  - 完整审阅修订
  - 批量签章与模板中心
  - 安全策略中心

- P2 交付
  - API/SDK 开放平台
  - 归档连接器
  - 多租户与运维看板

## 10. 关键风险与治理

- 标准解释差异导致兼容性不一致
  - 治理：建立规范解释白皮书与自动回归套件。
- 多方系统集成导致边界不清
  - 治理：接口契约先行，使用版本化 API 管理。
- 性能瓶颈集中在转换/验章链路
  - 治理：异步任务化 + 分级缓存 + 热点文档优化。
