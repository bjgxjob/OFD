# OFD GUI 工具技术路线与产品规划

## 1. 项目目标

开发一款面向桌面端的 OFD 文档工具，覆盖以下核心能力：

- 阅读：高性能查看 OFD，支持缩放、检索、目录跳转、批注查看。
- 创建：新建 OFD 文档与页面，支持文本、图片、图形对象插入。
- 存储：保存为 OFD 包（目录与压缩包两种形态），支持版本化存储。
- 转换：OFD <-> PDF 双向转换，支持“结构优先/视觉优先”模式。
- 编辑：按“文档 -> 页面 -> 内容块 -> 原子对象”精细编辑。

目标用户：

- 政企办公用户（阅读、盖章、归档）
- 文档处理人员（批量转换、纠错）
- 开发/集成用户（调用底层接口进行二次开发）

---

## 2. 产品形态与技术选型

### 2.1 产品形态

- 桌面应用（Windows/Linux，后续可扩展 macOS）
- 单机优先，支持离线使用
- 提供“GUI + CLI + SDK”一体化能力

### 2.2 推荐技术路线（主路线）

- UI 框架：`Qt 6 (Widgets + QML 混合)`
  - 原因：跨平台成熟、性能稳定、工程化能力强、中文输入与字体兼容成熟。
- 核心引擎：`libOFD`（现有 C/C++ 接口）
  - 直接复用已有读写、转换、签名、对象级编辑能力。
- 渲染策略：
  - 首选结构化渲染（文本/图片/PathObject）
  - 复杂场景回退视觉栅格（保持可视一致）
- 存储与配置：
  - 用户配置：`JSON`
  - 缓存与索引：`SQLite`
- 打包发布：
  - Linux：`AppImage / deb`
  - Windows：`MSI / NSIS`

### 2.3 备选路线（不优先）

- Electron + C++ Native Addon  
  优点：前端生态丰富；缺点：内存占用较高、原生渲染链路复杂。

## 3. 总体架构设计

采用分层架构，解耦 UI、业务、引擎：

1. **UI 展示层**
   - 主窗口、页面缩略图、对象属性面板、目录树、批注面板、任务中心
2. **应用服务层**
   - 文档会话管理、撤销重做、命令总线、转换任务编排、权限校验
3. **领域模型层**
   - 文档模型：Document/Page/Block/Object
   - 原子对象：TextObject/ImageObject/PathObject
4. **引擎适配层**
   - 封装 `libOFD` C API，提供稳定 Facade（统一错误码与日志）
5. **基础设施层**
   - 文件系统、缓存、日志、崩溃恢复、插件机制

## 4. 界面规划（信息架构）

### 4.1 主界面布局

- 顶部：菜单栏 + 工具栏（新建、打开、保存、导出、转换、签章）
- 左侧：文档结构树（文档/页面/内容块/对象）+ 目录书签
- 中央：页面画布（阅读与编辑）
- 右侧：属性面板（对象属性、字体、边界、CTM、资源绑定）
- 底部：状态栏（页码、缩放、坐标、任务进度、错误提示）

### 4.2 核心页面/弹窗

- 文件首页（最近文档、模板、批量转换入口）
- 转换向导（模式选择：自动/结构优先/视觉优先）
- 批注管理面板（筛选、定位、导出）
- 书签/大纲编辑器（层级调整、跳转目标设置）
- 表单设计器（后续版本）
- 权限与动作配置面板（Permissions / Actions）

## 6. 关键技术方案

### 6.1 文档模型与编辑命令

- 使用统一 `Command` 模式：每次编辑封装为命令对象。
- 支持撤销/重做（Undo/Redo），并记录变更快照。
- 对象采用稳定 `ID` 进行定位，避免仅靠索引导致错改。

### 6.2 渲染与性能

- 页面渲染缓存（当前页 + 前后 2 页预加载）
- 大文档分段加载（延迟解析）
- 缩略图后台线程生成
- 转换任务异步执行，UI 不阻塞

### 6.3 兼容性策略

- 结构化优先，复杂场景自动回退视觉栅格
- 字体策略：
  - 优先嵌入字体
  - 失败时启用字体替代映射
- 图像策略：
  - 统一资源编码与格式降级（JPEG/PNG/BMP）

### 6.4 错误处理与可观测性

- 错误码统一（映射 `libofd_status_t` -> UI 友好提示）
- 详细日志（转换链路、渲染链路、资源加载）
- 失败工单导出（日志 + 输入摘要 + 环境信息）

## 7. 非功能需求

- 启动时间：< 3 秒（典型机器）
- 100 页文档打开：< 2 秒（首屏可见）
- 常规转换成功率：> 98%
- 崩溃恢复：自动保存草稿
- 安全：本地敏感配置加密（证书路径/口令）

## 9. 测试与验收方案

### 9.1 测试类型

- 单元测试：对象解析、命令执行、错误码映射
- 集成测试：GUI 操作链路 + libOFD API
- 回归测试：真实样本库（签章/图片/中文字体/复杂路径）
- 兼容测试：WPS、主流 OFD 阅读器、不同操作系统

### 9.2 验收标准

- 关键功能全部可用：阅读、创建、保存、转换、编辑
- 对象级编辑正确率 >= 99%
- 真实样本转换通过率 >= 95%
- 无阻断级崩溃与数据损坏问题

---

## 10. 风险与应对

- **风险 1：不同阅读器兼容差异**  
  应对：建立兼容基线样本库，优先保证主流阅读器一致性。

- **风险 2：复杂文档渲染性能不足**  
  应对：分层缓存 + 异步渲染 + 降级策略。

- **风险 3：签章验签链路复杂**  
  应对：签章模块独立迭代，先保证可视一致，再完善验签报告。

- **风险 4：对象编辑导致结构破坏**  
  应对：编辑前后结构校验 + 自动备份 + 快速回滚。
